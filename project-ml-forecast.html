Demand Forecasting with Machine Learning

Predicting Q2 occupancy and revenue demand using Prophet time-series modeling.

Download Source Code

Moving Beyond Excel

Role: Data Scientist (Revenue)
Goal: Reduce forecast variance to <2%.

Traditional "Same Time Last Year" forecasting fails to account for complex market dynamics. I implemented a Machine Learning approach using **Facebook Prophet** to detect non-linear trends and seasonality (daily, weekly, yearly) automatically.

The Model

The model was trained on 24 months of historical booking data. It decomposes the time series into trend, seasonality, and holiday effects to produce a robust future prediction with confidence intervals. 

# Prophet Forecast Implementation
def forecast_demand(df):
    model = Prophet(yearly_seasonality=True, weekly_seasonality=True)
    model.fit(df)
    
    future = model.make_future_dataframe(periods=90)
    forecast = model.predict(future)
    
    return forecast[['ds', 'yhat', 'yhat_lower', 'yhat_upper']]

Strategic Value

The model predicts stabilized demand in April (contrary to the manual budget's drop), suggesting we should hold AIR (Average Initial Rate) steady rather than dropping prices for volume.
