Optimizing Portfolio Revenue: Bali Cluster Analysis

Strategic oversight of 3 Bali properties using real-time Amadeus API data and comparative analytics.

Download Source

The Business Context

Role: Cluster Revenue Analyst
Scope: 3 Properties (Resort, City Hotel, Boutique)
Data Source: Amadeus API (Cleaned & Anonymized)

Managing a multi-property portfolio requires moving beyond single-asset focus. The challenge was to identify performance disparities between our three assets (`Property_001`, `Property_002`, `Property_004`) to ensure the cluster as a whole outperforms the market share index.

Data Pipeline: Integrating Amadeus API

I engineered a pipeline to ingest real-time booking data from the Amadeus GDS. This allowed for granular analysis of 26,000+ booking records. The raw data was cleaned using Python to normalize currency and room class definitions across the disparate property management systems (Opera & Holidex). 

# Cluster Data Integration
def analyze_cluster_performance():
    # Ingesting Amadeus Data
    df = pd.read_csv("Data/hotel_data_cleaned.csv")

    bali_cluster = df[df['flg_region'] == 'Bali']
    
    # Aggregating Comparative Metrics
    cluster_adr = bali_cluster.groupby(['month', 'property_id'])['price_cleaned'].mean()
    
    return cluster_adr

Strategic Recommendations

Based on this data, I recommend the following actions to the Revenue Director:

  • Adjust Advance Purchase Rates: Given the heavy concentration of bookings in the 0-30 day window (Fig 2), we can tighten restrictions on "Early Bird" rates 45 days out to protect yield.
  • Premium Room Upsell: With "Deluxe" rooms driving a significant portion of revenue (Fig 3), we should focus marketing spend on upselling standard bookings to this category pre-arrival.
  • Seasonal Minimum Stay: Implement a 3-night MLOS (Minimum Length of Stay) during the Q3 ADR peak to maximize total booking value.
